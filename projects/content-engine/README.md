## Why this project exists

Τίποτα από τα υπόλοιπα projects δεν μονετίζεται χωρίς audience. Το content είναι το multiplier που κάνει όλα τα άλλα να αποδώσουν:
- Attracts freelance clients (highest short-term revenue)
- Validates product ideas before building
- Builds long-term defensibility (distribution > product στο AI era)

## Target audience choice

**Primary: Prosumers** (freelancers, SMB owners, creators, solo founders).
- Έχουν budget για tools/services
- Νοιάζονται για outcomes, όχι για code
- Μπορούν να γίνουν clients ή customers
- Μεσαίο audience size, υψηλό conversion

**Secondary: Devs/founders** (τεχνικό deep-dive content για X/YouTube long-form).

**NOT targeting mass audience** ως primary. Reach χωρίς purchasing intent = vanity.

## Core framing principle

Το τεχνικό δεν βαριέται ο κόσμος. Το **jargon** βαριέται.

- ❌ "Config ενός MCP server για Gmail API με OAuth"
- ✅ "Ο βοηθός μου διαβάζει τα mail μου και μου λέει τι προτεραιότητα έχει. Δες."

Κάθε video ξεκινάει από **outcome + story**, όχι από tech. Το plumbing κρύβεται ή πάει σε ξεχωριστό dev-focused video.

## Content pillars (rotate)

1. **Build logs** — "Σήμερα αυτοματοποίησα το Χ σε Υ λεπτά". Screen recording + voiceover. Πιο δυνατό format τώρα.
2. **Before/After** — 4 ώρες δουλειάς → 10 λεπτά. Split screen. Viral-friendly.
3. **"Έβαλα το AI να κάνει τη δουλειά μου για μια εβδομάδα"** — narrative series. Human-interest hook.
4. **Failures** — "Το agent μου έκανε Χ μαλακία και έχασα Υ ώρες". Authentic. Χτίζει trust.

**Mix:** 70% build/outcome, 20% lessons, 10% opinions/predictions.

## Concrete content angles

- **"Αυτοματοποιώ μια ολόκληρη εταιρεία μόνος μου με agents"** — core narrative. Innova solo + agent architecture = δυνατό story.
- **Build in public:** paramythas, youtube-insights, video-templates updates
- **MCP demos:** short demos για κάθε MCP server που στήνεις
- **"Claude manages my X για μια εβδομάδα"** — experiment series (Gmail πρώτα, είναι έτοιμο)

## Platform distribution (1 piece → 10+ posts)

- **YouTube long-form** (10-20min) = flagship. 1/εβδομάδα.
- **Shorts/TikTok/Reels** (30-60sec) = highlights από το flagship. 3-5/εβδομάδα.
- **X threads** = write-up + lessons. 2-3/εβδομάδα.
- **LinkedIn** = reformat του X thread. 1-2/εβδομάδα.

Κάθε flagship video πρέπει να παράγει ελάχιστο 10 posts σε άλλα platforms.

## Language strategy

- **Primary: English**. 100x market size. Τεχνικό audience είναι global.
- **Secondary: Greek** μόνο για LinkedIn και μόνο αν στοχεύουμε Ελληνικούς SMB πελάτες ρητά.

## Anti-patterns (μη κάνεις)

- Μην γίνεις "AI news channel". Commodity. Χάνεις.
- Μην κάνεις generic "top 10 AI tools" lists. Dead format.
- Μη μιλάς σε τεχνικό και μάζα στο ίδιο video. Χάνεις και τους δύο.
- Μη ξεκινάς με tool demos χωρίς outcome hook.
- Μη δημοσιεύεις asynchronously (χωρίς cross-post plan). 1 piece = 10 posts, πάντα.

## Production workflow (draft, refine as we go)

1. **Script/outline** — hook (0-3s), problem, build, outcome, CTA
2. **Screen recording** — QuickTime ή OBS, voiceover live
3. **Edit** — DaVinci / Descript / CapCut
4. **Thumbnail + title** — A/B test titles όταν μπορείς
5. **Cross-post** — από flagship σπάσε σε shorts + threads + LinkedIn posts

Link με `video-templates` όταν η νέα ιδέα που βασίζεται σε αυτό ωριμάσει.

## Success metrics (review monthly)

- **Output:** 4 flagship videos/μήνα (minimum)
- **Distribution:** 40+ cross-posts/μήνα
- **Audience (3 months):** 1k+ subs σε μία platform που μετράει
- **Revenue (6 months):** first paying client από inbound ή first product customer
